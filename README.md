# 工业机械故障智能问答系统

基于知识图谱 + 检索增强生成（RAG）的风电故障诊断一体化问答平台。系统融合 Neo4j 图数据库、云端大模型接口与轻量化检索架构，为运维人员提供可追溯、可解释的故障推理与维修决策支持。

---

## 功能概览

- 智能问答：支持风电故障诊断与日常对话双重模式，自动识别问题意图并分流处理
- 知识图谱可视化：基于 vis‑network 展示 Neo4j 故障关系网络，支持节点检索、双击引用与详情卡片
- 图谱优先的混合检索：故障问答先取知识图谱结构化证据，再用文档片段补充细节（支持 Chroma 向量库并可自动降级）
- 多模态输入：文本、语音、图片三种输入通道，预留缺陷检测模型接入接口
- 数据诊断：支持 CWRU / MFPT 数据集，集成 1D‑CNN、WDCNN、CNN‑LSTM、CNN‑Transformer 四种预训练模型进行轴承故障推理
- 双重知识库：Markdown 文档 + CSV 结构化数据共同构成 RAG 文本库，支持管理员上传 CSV 扩充
- 用户体系：区分管理员与普通运维工程师，提供注册登录、个人中心、头像上传与密码修改
- 管理员后台：控制台统计（用户/节点/案例）、图谱管理（CSV 上传/检索/筛选/分页）、诊断案例管理（增删改查/导入导出）

## 环境要求

- Python：3.9+（建议 3.12，需与本地 torch 版本兼容）
- Neo4j：Neo4j AuraDB（云端免费实例）或本地 Neo4j Community Edition
- 大模型 API：SiliconFlow（硅基流动）API Key（免费额度即可）
- 可选依赖：`chromadb`（向量检索，不安装则自动降级为关键词匹配）；`numpy / scipy / torch`（数据诊断模块，仅需在使用该功能时安装）

## 安装与运行

### 本地部署

1. 克隆项目并进入目录：

   ```bash
   cd wind-cloud-0504
   ```

2. 创建虚拟环境并激活（Windows PowerShell 示例）：

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   ```

3. 安装依赖：

   ```bash
   pip install -r requirements.txt
   ```

4. 配置环境变量（可选，建议通过系统环境变量或 `.env` 文件设置）：

   ```powershell
   $env:NEO4J_URI="neo4j+s://<your-aura-instance>.databases.neo4j.io"
   $env:NEO4J_USER="<neo4j用户名>"
   $env:NEO4J_PASSWORD="<neo4j密码>"
   $env:NEO4J_DATABASE="<neo4j数据库名>"
   $env:SILICONFLOW_API_KEY="sk-你的SiliconFlow密钥"
   $env:SILICONFLOW_MODEL="Qwen/Qwen3-8B"
   ```

   > 若未设置环境变量，系统将使用代码中内置的默认值（可用但建议替换为自己的凭证）。

5. 启动服务：

   ```bash
   python app.py
   ```

6. 浏览器访问：`http://127.0.0.1:7860`

### 云端部署

项目已适配 Railway 等平台，无需信用卡即可部署：

1. 将项目推送至 GitHub 公开仓库。
2. 在 Railway 中新建 Web Service，连接该仓库。
3. 设置启动命令 `python app.py`，并在 Variables 中添加上述环境变量。
4. 部署成功后获得公开访问链接（目前本项目已部署至：`https://web-production-5eb934.up.railway.app`）。

## 典型使用流程

### 普通运维工程师

1. 登录：使用已注册的普通用户账号登录（默认用户名/密码：`11 / 123`）。
2. 首页：查看系统概览与功能导航。
3. 智能问答：
   - 日常问候直接由云端大模型回复。
   - 输入风电故障问题（如“齿轮箱油温过高怎么办？”），系统自动走“图谱+检索增强”链路，返回结构化诊断结论。
4. 图谱可视化：
   - 浏览设备‑故障‑维修关系网络。
   - 双击节点可将其引用至问答区，结合图谱证据辅助提问。
   - 单击节点点击查看详情，可查看 CSV 证据卡片，包含故障描述、原因、维修记录等。
   - 可检索关键词（如“齿轮箱”、“油温”），返回相关节点及对应的三元组。
5. 数据诊断：
   - 选择数据集（CWRU / MFPT）、诊断模型（CNN / WDCNN / CNN‑LSTM / CNN‑Transformer）。
   - 选择 .mat 样本文件，点击诊断，利用所选模型在线推理，获取故障类型、置信度及振动波形/频谱图。
6. 个人中心：修改用户名、联系方式、头像及密码。

### 管理员

1. 登录：使用管理员账号登录（默认用户名/密码：`admin / 123456`）。
2. 控制台：查看用户总数、图谱节点数、案例数，并可视化展示 Top10 故障部件与频率分布。
3. 图谱管理：
   - 上传 CSV 文件扩充知识库；支持按类型/关键词检索筛选。
   - 查看任意 CSV 文件内容明细。
4. 诊断案例管理：
   - 手动新增案例，填写故障位置、关联、后果、来源。
   - 导入 CSV 批量添加案例；支持编辑、删除、搜索与分页浏览。
   - 可根据故障位置、关联设备、后果等字段进行筛选。
   - 可分页查看案例列表，每页显示 10 条记录。
5. 同样具有图谱可视化、智能问答、数据诊断功能。

## 主要模块说明

| 模块 | 技术实现 | 说明 |
|:---|:---|:---|
| 意图识别 | 关键词匹配 + 正则 | 区分日常对话与故障诊断，分流处理 |
| 知识图谱 | Neo4j + Cypher | 存储设备、故障、原因、维修等实体及关系 |
| 混合检索 | 图谱优先 + Chroma / 轻量关键词 | 图谱结构化证据 → 文档补充 → 大模型总结 |
| 大模型调用 | SiliconFlow API (OpenAI兼容) | 工业故障问答与日常对话生成 |
| 数据诊断 | PyTorch + 预训练模型 | 一维振动信号分类，输出故障类型与置信度 |
| 用户管理 | SQLite + werkzeug 加密 | 注册、登录、角色权限、头像上传 |
| 案例管理 | SQLite + CSV 导入 | 故障案例增删改查、分页检索与批量导入 |
| 前端界面 | 原生 CSS + vis‑network | 拟态设计、响应式布局、节点交互自封装 |

## 可扩展方向

1. 向量检索升级：使用更先进的RAG模型，提升语义召回率，降低模型幻觉，确保回复真实准确。
2. 图像诊断：接入 YOLO/SegFormer 等视觉模型，实现叶片裂纹、油污等缺陷的自动检测。
3. 语音识别：增加离线 ASR 服务（Whisper/FunASR），适应风电场弱网环境。
4. 维护预测：引入时序模型（LSTM/TCN），结合 SCADA 数据预测设备剩余寿命与告警。
